# 优化外来词汉字选取方案（以“鲁棒性”在中国为例）

## 目标
为来自其他语言的学术与技术词汇制定可重复的汉字选取流程，兼顾**语音贴合、语义准确、书写习惯、可检索性**，并以“robustness”在中国常译为“鲁棒性”为起点。

## 核心原则
1. **语义透明度**：优先选择能直接呈现核心含义的字。含义偏差超过 20% 时须用释义型译法而非音译。
2. **音韵贴合度**：根据普通话音节最短路径匹配外语发音，控制声母/韵母偏离在 1 步以内（如 /r/→/l/）。
3. **语义-音韵折衷评分**：S = 0.55×语义分 + 0.30×音韵分 + 0.10×书写频率分 + 0.05×负面联想惩罚（负值）。
4. **语域一致性**：科技、法律等专业领域允许较低口语友好度，但需提供标准释义句。
5. **多字组合的连贯性**：组合内的字频率中位数应不低于 10⁻⁶（基于现代汉语语料）以降低识读难度。
6. **跨区域一致性**：优先沿用两岸三地已有译名；若差异大，提供主译名与备选名并列。
7. **检索与信息系统兼容性**：避免生僻字、异体字；新增译名需绑定英文原词与领域标签便于索引。

## 标准化流程
1. **语义剖析**：拆解原词的核心语义与用途，形成 1–2 个关键词（例：“稳定”“抗干扰”）。
2. **语音映射**：用普通话音节库生成 3–5 个音译候选。
3. **字频筛选**：过滤掉语料频率过低或负面联想的字形。
4. **组合打分**：对候选组合按 S 评分；语义权重最高。
5. **语境验证**：在 3–5 个典型句中试用，检查歧义与流畅度。
6. **发布与维护**：记录译名、评分、释义句、领域标签、来源文献，并建立版本号。

## “鲁棒性”的示例评估（领域：计算机科学/工程）
- **语义剖析**：系统对扰动不敏感、性能稳定。
- **音韵映射**：`robustness` → /roʊˈbʌst/ ≈ “鲁/露/若” + “棒/邦/邦” + “性”。
- **候选组合**：
  - 语义型：稳健性、坚牢性。
  - 音义兼顾：鲁棒性、露邦性。
- **评分示例**（满分 10）：
  - 鲁棒性：语义 8（“棒”含坚固），音韵 6，字频 7，惩罚 0 → **S ≈ 7.0**。
  - 稳健性：语义 9，音韵 3，字频 8，惩罚 0 → **S ≈ 7.0**。
  - 坚牢性：语义 8，音韵 2，字频 7，惩罚 0 → **S ≈ 6.0**。
- **推荐策略**：
  - 技术文档沿用“鲁棒性”，并在首处给出英文 `robustness` 说明以保留跨文献一致性。
  - 面向公众或政策文本可用“稳健性”，提升语义直观度；必要时并列“（鲁棒性）”。

## 可操作指南
- 为新外来词制作**候选表**（含语义关键词、音节、评分、语域）。
- 建立**标准句模板**：
  - 技术：`该模型在噪声输入下保持 <译名>。`
  - 普及：`系统在受到干扰时仍然 <译名>。`
- 运行**年度回溯评估**：
  - 收集实际使用频率与误解案例。
  - 若语义偏差 >10% 或读者困惑度显著上升，调整主译名或释义。

## 展望
- 构建开放数据库，记录外来词的译名、评分、领域标签、语境示例。
- 引入**跨语料自动打分**（结合词向量和语音距离），辅助人工审定。
- 与学术社群、标准化机构协作，形成可追溯的更新机制。
