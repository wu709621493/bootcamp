# 广州市地铁应急和灾害管控指挥中心方案

## 建设目标
- 建立覆盖 **预防、监测、处置、恢复** 的全周期指挥体系，提升极端天气、设备故障、客流暴增和公共安全事件的响应速度。
- 以“数据驱动 + 现场联动”为原则，统一调度行车、客运、设施、公安、消防、医疗、城管等多部门资源。
- 为退役安检体系提供平稳退出与升级替换的标准流程，确保服务不中断、数据不丢失、责任有闭环。

## 系统架构
1. **感知层**：车站监控、隧道环境传感器（烟雾、温湿度、有害气体）、轨旁设备状态监测、列车载荷及速度传感器、客流计数器、票务闸机、安检设备与 RFID 箱包标签。
2. **通信层**：地铁自有专网 + 城市应急专网互联，支持 5G/MEC 边缘节点，将高频视频流和传感数据就近计算后再汇总。
3. **数据层**：实时数据总线（Kafka/ Pulsar）、时序数据库用于传感数据、数据湖用于历史事件与视频归档；统一主数据涵盖车站、线路、设备、人员、合作机构。
4. **应用层**：
   - **态势感知与数字孪生**：3D 线路与车站模型，叠加客流热力、列车间隔、环境指标、风险点。
   - **事件管理**：事件模板库（火灾、淹水、轨障、安检报警、设备失效、公共卫生事件等），自动生成处置清单、责任到人和时限。
   - **联动指挥**：一键调度列车越行、清客、限流、广播、告示板推送；接口对接公安消防 110/119、医院急救 120 与城市应急平台。
   - **资源管理**：备品备件、救援车辆、移动排水泵、应急发电车、便携通信设备等库存与位置管理。
   - **评估与复盘**：事件后自动生成报告，包含时间线、语音记录、视频片段、 KPI 对比和改进建议。
5. **安全与合规**：零信任接入、最小权限、操作留痕审计；敏感视频与乘客信息分级加密；灾备双活与演练计划。

## 指挥流程（示例）
1. **触发**：传感器或人工报送触发事件；系统按规则匹配事件类型与等级。
2. **研判**：事件看板汇总证据，自动推荐处置方案；值班长与专业人员在线确认。
3. **指令下发**：向车控、车站、区间设备、合作机构同步任务；现场移动端接收并回传状态。
4. **执行跟踪**：看板显示关键节点完成率；未按时节点触发升级或替补资源。
5. **恢复与复盘**：确认行车图恢复、客流疏散完成、设备检修闭环；生成复盘报告并进入培训库。

## 退役安检体系方案
- **现状盘点**：梳理各线安检设备型号、安装点、维护状态、生命周期与合同义务，明确需退役与需保留的清单。
- **风险评估**：对退役窗口期的客流高峰、重大活动、治安风险进行评估，确定分阶段替换顺序与临时保障措施。
- **数据与接口迁移**：
  - 归档安检历史记录（报警、影像、告警处置日志），对涉案数据按法规加密存储并保持追溯链路。
  - 新旧系统之间建立临时双写或镜像模式，确保闸机、车站监控、票务系统接口在切换期间无缝。
- **设备处置与资源再利用**：
  - 对可继续使用的模块（X 光机/金属探测门部件、电源、网络交换单元）进行检测和二次利用。
  - 对报废设备按环保与数据安全规范进行拆解、去标识和销毁。
- **人员与流程调整**：
  - 对安检岗位人员进行转岗培训（客流疏导、巡站、客服应急），明确新的岗位职责与绩效指标。
  - 更新应急预案和现场 SOP，去除或替换对旧安检流程的依赖；重新测试和演练。
- **上线验收与过渡**：制定分站点、分时间段的切换计划，设置回退方案；通过联合演练和第三方安全测评后正式退役旧系统。

## 关键指标（示例）
- 事件自动识别准确率 ≥ 95%，指令下发到执行确认平均时延 < 30 秒。
- 重大事件处置按时率 ≥ 98%，跨部门联动响应时间减少 30% 以上。
- 安检退役切换期间客流延误控制在 3 分钟以内，无安保等级下降事件。
- 数据归档完整率与接口可用率均 ≥ 99.9%。

## 推进路线图
1. **0-3 个月**：需求梳理、数据治理、设备/接口清单；搭建事件管理与可视化最小可用产品（MVP）。
2. **3-6 个月**：多源数据接入（传感器、视频、票务、安检）、联动指令平台、应急预案库与演练工具上线；开展安检退役试点。
3. **6-12 个月**：全面推广、双活与灾备部署、智能研判与预测模块；完成安检体系平稳退役与人员转岗。
4. **持续运营**：季度复盘、年度安全评估与外部审计，迭代优化算法与预案库。
