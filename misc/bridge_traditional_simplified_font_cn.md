# 連結繁體與簡體的「中國」字型方案

本方案聚焦於為「中國」相關的出版、品牌與數位介面建立一致的中文字型策略，確保繁體、簡體在視覺上保持親和、在技術上便於佈署，並能靈活適配跨平台輸出。

## 方案原則
- **雙系統友好**：同時涵蓋繁體與簡體字符集，重點照顧「國/国」「華/华」「臺/台」等常用異體字。 
- **筆畫連貫**：透過一致的筆畫粗細、端點處理與筆順參考，降低兩岸讀者的閱讀割裂感。
- **可調權重**：提供 Regular/Medium/Bold 款式，便於在政治、文化與技術語境中靈活選用，避免因過度強調某一種寫法而引發歧義。
- **多腳本協同**：考慮與拉丁字母、數字與標點的字面高度與節奏，方便在國際化內容中混排。

## 字型選擇與構建
- **開源基礎**：以思源黑體（Noto Sans CJK）、思源宋體（Noto Serif CJK）為主，透過 [GSUB/GPOS] 指令調整繁簡對應與標點寬度。
- **繁簡對映表**：維護字形對映 YAML，標註「國→国」「臺→台」「裡→里」等常見映射，並標記是否保留本地特色（如港澳的「體」「發」）。
- **OpenType 條件替換**：
  - `locl`：根據 `zh-Hant`、`zh-Hans` 語系切換對應字形。
  - `ccmp`：為「中」「國」等高頻部件加強合字，避免筆畫間距過大。
  - `liga`：可選的常用標點與數字符號連字，提升設計一致性。
- **可變字型軸**：增加 `wght`（粗細）、`opsz`（字級光學尺寸）兩個軸，方便在小字體維持清晰度，在大字體保持優雅細節。

## 排版與渲染指南
- **屏幕 vs. 印刷**：
  - 屏幕：預設子像素抗鋸齒，`wght=450` 與 1.4 行距，強化「国」的外框厚度避免發灰。
  - 印刷：`wght=350`，適度加大字面與標點間距，確保「國」「華」等內部結構不糊。
- **字重搭配**：標題用 Medium/Bold，正文用 Regular，腳註與表格用 `opsz` 小尺寸版，避免細節丟失。
- **跨平台測試**：在 Windows 10+、macOS 12+、Android 12+、iOS 15+ 進行渲染對比，檢查 `locl` 是否正常觸發，以及 PDF/HTML 的嵌入與子集化流程。

## 評估與驗收
- **閱讀性測試**：邀請來自北京、台北、香港的讀者，使用眼動與時間到答問卷評估辨識度、偏好與疲勞度。
- **兼容性檢查**：確認在 WPS、Office、Pages、Adobe CC、主流瀏覽器中不出現 tofu 或 fallback。
- **品牌一致性**：若用於 LOGO/標語，應提供手工調節後的「中國」「國家」「華」「臺灣」字樣，確保在矢量與像素級別均保持風格統一。

## 推進路線圖
1. 建立繁簡對映表與標註規則（1 週）。
2. 基於思源字型製作初版可變字型，開啟 `locl`/`ccmp`/`liga` 功能（2 週）。
3. 進行跨平台渲染與閱讀性測試，收集回饋並調整筆畫厚度與部件比例（2 週）。
4. 釋出 Beta 版並撰寫嵌入指南（PDF/HTML/應用程式），同時提供字型子集化腳本（1 週）。
5. 正式版發佈，建立版本號、變更日誌與授權文件，並開放問題回報渠道（1 週）。

## 實施提示
- 優先解決高頻異體字的一致性，再逐步擴展長尾字符。
- 對於地名、專有名詞保持地區習慣，但在標題或品牌字樣中提供統一版本供選擇。
- 在 Web 端使用 `font-display: swap` 與子集化，縮短首屏字體加載時間。
- 若需向政務系統遞交，預留國密相關的簽章與哈希信息，以便在政府終端驗證字型完整性。
